Tor: Entwickler

Blättere durch das CVS Repositorium von Tor: (das muss nicht notwendigerweise funktionieren oder kompilieren)

  • CVS Sandbox wird regelmäßig auf neuesten Stand gebracht
  • ViewCVS
  • anonymer pserver-Zugang (Passwort ist: guest):
    • Lege ein neues Verzeichnis an und wechsele hinein.
    • cvs -d :pserver:guest@cvs.seul.org:/home/or/cvsroot login
    • cvs -d :pserver:guest@cvs.seul.org:/home/or/cvsroot co tor
    • cd tor; ./autogen.sh; make; make install if you like.
    • (nutze -r tor-0_0_9 oder etwas ähnliches, um eine bestimmte Version zu beziehen.)
    • (Um die Maintenancebranch auszuchecken, nutze -r tor-0_0_9-patches)

Hier ist die aktuelle Roadmap für die Version 0.1.1.x:

  • CPU-Last auf den Servern reduzieren.
  • Das Verzeichnis weiter dezentralisieren:
    • Mehr permanente Verzeichnisserver sammeln und ihre Schlüssel in den Code packen.
    • Es muss geklärt werden, was 'verified' bedeutet: Es bedeutet, dass der Nickname registriert ist.
    • Eine Methode für Clients, die Anzahl der Server in einer sicheren Weise zu partitionieren: So müssen sie nicht alle lernen und sind nicht so leicht partitionierbar. Idee aufschreiben, aber wahrscheinlich noch nicht umsetzen.
  • Helferknoten (zumindest vorläufig).
  • Enklaven (zumindest vorläufig).
  • Einen GUI-Wettbewerb ins Leben rufen.
  • Irgendwas für den Systray in Windows.
  • Webseiten, die auf Tor verweisen: indymedia.org. andere?
  • Forschung: Skalierbarkeit, an Ende-zu-Ende-Attacken denken.

Liste von Sachen, die noch von den Entwicklern erledigt werden müssen.
